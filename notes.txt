It is often used to control source code by programmers collaboratively developing software.

branch lets you work on features or fixes without affecting the main code.

git branch -> see all branches

merge conflict
happens when git cannot automatically combine changes from two branches
when? : when two branches changed the same oart of same file and git doesnt know which one to keep
example:
<<<<<<< HEAD
Hello World
=======
Hello Git
>>>>>>> bug-fix
HEAD → current branch
bug-fix → incoming branch

how to fix:
git status (will show files with conflict)
resolve manually (choose one or combine)
git add file.txt
git commit (mark as resolved)

if completely messed up -> abort the merge
git merge --abort

fix:
git stash
git stash temporarily saves your uncommitted changes (both staged + unstaged)
so you can switch branches or pull code without committing.

git stash → Temporarily saves your current uncommitted changes and cleans your working directory.
git stash list → Shows all saved stashes.
git stash apply → Applies the latest stash without deleting it.
git stash apply stash@{n} → Applies a specific stash by its index.
git stash branch <branch-name> stash@{n} → Creates a new branch and applies the specified stash to it.
git stash pop → Applies the latest stash and removes it from the stash list.
git stash drop → Deletes a specific stash.
git stash clear → Deletes all stashes permanently.



tag:
git tag is used to mark specific commits with a label (usually for versions like v1.0).
git tag = assign a meaningful name to a commit (e.g., version release)

git tag → Lists all tags in the repository
git tag v1.0 → Creates a lightweight tag on current commit
git tag -a v1.0 -m "version 1.0" → Creates an annotated tag with message
git tag -d v1.0 → Deletes a tag locally
git push origin v1.0 → Pushes a specific tag to GitHub
git push origin --tags → Pushes all tags to GitHub


git rebase:






git remote -v → Shows all remote repositories linked to your local repo (with URLs).
git remote add origin <repo link> → Connects your local repo to a remote repo named origin.
git push origin main → Pushes your local main branch to the remote origin.

git push -u origin main → Pushes branch and sets it as upstream (so future git push works without specifying branch).
Pushes your branch to remote
Links your local branch (main) with remote branch (origin/main)
origin = remote repo name
-u = set upstream (link local ↔ remote branch)
